回報者指出在 Hermes Desktop 中，即使在 Settings 中正確設定了新的 Working Directory 並寫入 config.yaml 的 terminal.cwd，若 renderer 端的 localStorage 已記住舊的 workspace 路徑，新 session 仍會在舊目錄啟動，造成 UI 顯示已設定新路徑但實際行為不一致。
