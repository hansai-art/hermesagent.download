提案指出 Hermes 目前每個 agent 都需要獨立的 profile 與 gateway 行程，若要同時跑多個不同用途的 agent 就要開多個 process，各自佔用資源；提案參考 OpenClaw 的單一 daemon 架構，讓多個 agent 共用同一個 daemon 行程，各自擁有獨立 workspace 與記憶，並依 session key（含群組內 topic）路由訊息。
