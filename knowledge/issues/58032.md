把 gateway.multiplex_profiles 從 true 改回 false 後，先前在多工模式下建立的 profile session 仍殘留在共用的 state.db 中，gateway 會持續讀到這些過期 session 並把訊息路由到錯誤的 profile，即使該 profile 的 gateway 已停用。
