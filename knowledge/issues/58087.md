回報者指出當 agent 產生很短但完整的最終回應時，對話迴圈的 finish_reason == length 截斷處理邏輯仍會誤判並注入「回應被截斷，請繼續」的系統訊息，導致 agent 在 Telegram 上重複傳送同一則簡短回應 2 到 3 次。
