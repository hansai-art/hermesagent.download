此 issue 指出 Ollama Cloud（OpenAI 相容 API 的雲端推論服務）目前只能透過 custom_providers 設定使用，這會造成幾個問題：provider: ollama-cloud 在 auxiliary 區塊不會被辨識、需要在每個 auxiliary 任務設定中重複寫 provider: custom 加上 base_url 與 api_key、而且 auth.py 裡的 PROVIDER_REGISTRY 並未包含 ollama-cloud。提案是把 ollama-cloud 加進 PROVIDER_REGISTRY，並在 auxiliary_client.py 設定對應的預設 auxiliary model。此 issue 已關閉。
