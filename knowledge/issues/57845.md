回報者指出在 OpenRouter/envelope 快取版面下，system_and_3 策略把快取斷點放在 tool 訊息與內容為空的 assistant 訊息上，但這兩種訊息實際上無法有效承載快取標記，導致 agent 迴圈中除了 system prompt 之外的快取斷點全部靜默失效，整段對話每次都以全額輸入價格重新計費。此 issue 已關閉。
