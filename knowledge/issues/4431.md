這個 issue 指出目前用同一個 gateway 跑多個 Telegram 群組或 Discord 伺服器時，所有 session 都共用同一組人格、system prompt、CLAUDE.md 與工作目錄，無法針對不同群組或討論串客製化。提案在 config.yaml 新增 topic_configs 區塊，依平台與聊天室 ID 對應各自的設定覆寫。
