回報者指出使用 DeepSeek 等預設啟用 thinking 區塊的模型時，agent 的文字回應無法傳送給使用者，只會顯示終端機工具的輸出結果；在 cron/心跳排程情境下，因使用者看不到回應而無法給出新指示，導致 agent 反覆觸發、耗費大量 API 費用並形成無限迴圈。
