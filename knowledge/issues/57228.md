長時間執行的 Hermes worker 會累積孤兒 MCP stdio 子程序，案例中單一 worker process 累積了 53 個 mimir 子程序，共佔用 1.4GB RSS 與大量檔案描述符，全部搶同一個 SQLite DB，超過約 50 個子程序後會造成 DB handle 爭用，導致 memory 工具呼叫間歇性失敗，但 hermes mcp test mimir 仍顯示健康。
