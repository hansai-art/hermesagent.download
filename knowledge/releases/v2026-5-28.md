如果你的 Hermes 用久了、session 一多,搜尋歷史對話開始變慢——這一版把那個痛點直接解決了,而且順便幫你省錢。

2026 年 5 月 28 日發布的 **v0.15.0(代號 Velocity)**,主軸是「把引擎拆開、讓每一塊都更快」[^1]。以下挑對中文使用者最有感的幾項。

## session 搜尋快了 4500 倍,而且不再花錢

以前 `session_search`(在歷史對話裡找東西)是靠 LLM 去理解你要找什麼——慢,而且每次搜尋都要算錢。這一版把它**重寫成不呼叫 LLM 的版本**:快了約 **4500 倍,而且零成本**[^2]。

對於累積了大量對話、常常需要「我之前是不是問過類似的事」的人,這是有感的日常改善。

## 冷啟動繼續變快

延續上一版的提速,這一版**再省下約一秒**的冷啟動時間,每一輪的函式呼叫也少了約 47%[^1]。累積下來,`hermes` 啟動與每次回應都更俐落。

## 主程式終於不再是 16,000 行

`run_agent.py` 這個核心檔案以前有一萬六千行——這一版做了「大重構」,把它拆開[^1]。

這件事對日常使用者沒有直接可見的效果,但它是後續所有功能能更快迭代的基礎。**如果你是想貢獻上游程式碼的開發者,這一版之後的 codebase 好讀很多。**

## 擋下 prompt 注入攻擊

新增了針對 **Brainworm 這類 prompt 注入攻擊**的防禦,在三個關卡攔截[^1]。

如果你讓 agent 讀取外部內容(網頁、別人傳來的檔案、email),這類攻擊會試圖用藏在內容裡的指令劫持 agent。這一版強化了對這種攻擊的抵抗。

## 一把 token 取代所有 API key

新增 **Bitwarden Secrets Manager** 支援:用一把 bootstrap token,取代原本每個供應商各自一把的 API key[^1]。

機密集中管理,不用在設定檔裡散落一堆 key。

## 新的訊息平台與功能

- **ntfy** 成為第 23 個訊息平台——推播通知,**不需要註冊帳號**[^1]
- **Skill bundles**:用 `/<名稱>` 一次載入多個 skill,不用一個一個開
- **TUI session 協調器**:在同一個 TUI 視窗裡同時開多個即時 session
- 兩個新的生圖供應商:Krea 2(Medium + Large),FAL 改成 plugin

## 這一版對你既有的設定有影響嗎

大多是新增與提速,不需要改動現有設定。但兩點值得注意:

1. **`session_search` 的行為變了**——它不再用 LLM,所以搜尋的「理解能力」可能跟以前不同(換來的是速度與零成本)。如果你依賴模糊語意搜尋,留意一下結果差異。
2. 如果你之前為了省 session 搜尋的錢而避免使用它,現在可以放心用了。
